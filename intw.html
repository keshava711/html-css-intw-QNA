Tags are the fundamental building blocks of HTML, used to define and structure the content of a webpage. They are enclosed within angle brackets (<>) and typically come in pairs, with an opening tag and a closing tag.
There are two types of Tags
  • paired tag
  • unpaired tags

    • Paired tags in HTML are tags that come in pairs: an opening tag and a closing tag. The opening tag marks the beginning of an element, while the closing tag indicates its end.

    Unpaired tags in HTML are tags that do not have a corresponding closing tag. They are typically used to represent self-closing elements.

    Git is a free, open-source version control system that helps software developers collaborate on projects. It's the most widely used version control system in the world and is considered the modern standard for software development.

    Git is a version control system that lets you manage and keep track of your source code history. GitHub is a cloud-based hosting service that lets you manage Git repositories. If you have open-source projects that use Git, then GitHub is designed to help you better manage them.

    A local repository is a copy of the entire project's history and codebase that resides on a developer's machine. When a developer initializes a local repository, Git sets up the necessary infrastructure to track changes, branches, and commits within the project.
    In Git, a remote repository is a version of a project that's hosted on a network or the internet and shared among multiple team members. Remote repositories can be located on a local machine, but the term "remote" doesn't necessarily mean that the repository is on a different network or internet location

    Initializing a new repository: git init

    1. Cloning a Repository: Definition: Creating a local copy of a remote Git repository.
    Advantages:
    Full control over the code's structure, logic, and implementation. Potential for a deep understanding of the code's functionality. Tailored to specific needs and requirements.
    Disadvantages:
    Time-consuming and resource-intensive. Requires strong programming skills and expertise. May introduce errors or inefficiencies if not carefully crafted.
    2. Using Submodules: Definition: Incorporating external Git repositories as subdirectories within your main project. Advantages: Saves time and effort by avoiding reinventing the wheel. Access to well-tested and optimized code. Encourages code reuse and modularity.
    Disadvantages:
    May require learning new APIs or syntax. Can introduce dependencies and potential compatibility issues. May not always provide the exact functionality or customization needed.

    1.Initialization: Create a new Git repository using git init in your project's directory.
    2.Staging: Add changes to the staging area using git add filename. This prepares the files for commit.
    3.Committing: Create a snapshot of the staged changes using git commit -m "Commit message". This saves the changes to your local repository.
    4.Pushing: Upload your local commits to a remote repository using git push remote branch. This makes your changes accessible to others.

    Git status is a command used in Git to check the current state of your working directory and staging area. It provides information about which files have been modified, added, deleted, or staged for commit.
    Key information provided by git status:
    Branch: The current branch you are working on.
    Untracked files: Files that are not being tracked by Git.
    Changes not staged for commit: Modified files that have not been added to the staging area.
    Changes to be committed: Files that have been added to the staging area and are ready to be
    committed.

    branch-name with the desired name for your new branch.

    To see local branches, run this command: git branch.
    To see remote branches, run this command: git branch -r.
    To see all local and remote branches, run this command: git branch -a.
